# 需求文件

## 簡介

Quick Text Copy 是一個 Chrome 瀏覽器擴展，讓使用者能夠快速將當前頁面的標題和網址複製到剪貼簿，格式適合分享。此擴展提供一鍵解決方案來擷取頁面資訊，無需手動選取或格式化。

## 需求

### 需求 1

**使用者故事：** 作為網頁使用者，我希望能夠一鍵快速複製當前頁面的標題和網址，以便我能夠以一致的格式輕鬆分享頁面資訊。

#### 驗收標準

1. 當使用者點擊擴展按鈕時，系統應將當前頁面的標題和網址以「標題 網址」格式複製到剪貼簿
2. 當複製操作完成時，系統應不提供視覺回饋（靜默操作）
3. 當使用者貼上複製的內容時，系統應提供確切格式「標題 網址」的文字，以單一空格分隔

### 需求 2

**使用者故事：** 作為網頁使用者，我希望擴展能在所有標準網頁上運作，以便我能在不同網站上一致地使用它。

#### 驗收標準

1. 當使用者在 HTTP 或 HTTPS 網站上時，系統應成功複製頁面資訊
2. 當使用者在 chrome:// 或 chrome-extension:// 頁面上時，系統應跳過操作且不產生錯誤
3. 當頁面標題或網址包含特殊字元時，系統應在複製的文字中正確處理它們
4. 當頁面標題超過 150 個字元時，系統應以「...」後綴截斷它

### 需求 3

**使用者故事：** 作為網頁使用者，我希望擴展能在不同剪貼簿情境下可靠運作，以便我能在各種環境中依賴它。

#### 驗收標準

1. 當現代剪貼簿 API 可用時，系統應使用 navigator.clipboard.writeText()
2. 當剪貼簿 API 失敗或不可用時，系統應回退到 document.execCommand('copy')
3. 當兩種剪貼簿方法都失敗時，系統應記錄錯誤而不崩潰
4. 當複製操作超過 2 秒時，系統應優雅地逾時

### 需求 4

**使用者故事：** 作為開發者，我希望有完整的日誌記錄和錯誤處理，以便我能夠排除問題並監控擴展效能。

#### 驗收標準

1. 當任何操作發生時，系統應將適當的除錯資訊記錄到控制台
2. 當錯誤發生時，系統應記錄詳細的錯誤資訊與上下文
3. 當擴展啟動時，系統應記錄 service worker 初始化
4. 當操作完成時，系統應記錄時間資訊以進行效能監控

### 需求 5

**使用者故事：** 作為網頁使用者，我希望擴展具有適當的視覺識別和中繼資料，以便我能在瀏覽器中輕鬆識別和管理它。

#### 驗收標準

1. 當擴展安裝時，系統應顯示適當的圖標，尺寸為 16px、32px、48px 和 128px
2. 當使用者將滑鼠懸停在擴展按鈕上時，系統應顯示工具提示「複製頁面資訊」
3. 當擴展出現在擴展列表中時，系統應顯示名稱「Quick Text Copy」和適當的描述
4. 當擴展載入時，系統應僅需要 activeTab 和 scripting 權限
